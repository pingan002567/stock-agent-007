# 前端可视化交互设计

## 总体定位

前端不是聊天机器人页面，而是一个“投研操作台 + AI Copilot”。用户主要围绕自选、持仓、个股、市场、盯盘事件工作，AI 始终在右侧提供解释、追问、规划和任务入口。

## 布局

当前原型采用三栏布局：

```text
左侧菜单栏    中间内容栏                 右侧 AI Copilot
74px fixed   主工作区，独立滚动          约 1/3 宽，可关闭/展开
```

要求：

- 菜单栏、内容栏、AI Chat 各自拥有滚动条。
- 设置入口固定在左侧菜单底部，用齿轮图标表示。
- 右侧 AI Copilot 是全局面板，不放进某个页面内部。
- AI Copilot 支持关闭和展开；关闭后中间工作区扩展。
- 顶部保留状态条、全局搜索、数据/权限状态，不再放“运行 AI 指令”按钮。

## 页面职责

### 总览

总览是首页，不叫“指挥”。它汇总用户当前最关心的信息：

- 大盘信息。
- 账户/组合概览。
- 持仓摘要。
- 自选摘要。
- 盯盘事件摘要。
- 当前研究对象。
- 调仓方案草案和影响模拟。
- 任务和审计摘要。

总览不展示 AI Skills 状态。AI Skill 是背后的能力和设置项，不应抢占首页注意力。

### 自选

自选是用户主动关注的股票池：

- 自选列表。
- 分组和标签。
- AI 关注理由。
- 加入/移除自选。
- 从自选进入个股深研。
- 将自选纳入盯盘规则。

### 持仓

持仓是用户真实资产视角，原“组合”命名不够直观，因此改为“持仓”：

- 持仓列表。
- 仓位占比。
- 盈亏和风险状态。
- 集中度、行业暴露、波动风险。
- 持仓导入。
- 风险扫描。
- 进入调仓规划。

### 个股

个股页面承载搜索结果和单股信息：

- 全局搜索支持代码、名称、拼音。
- 搜索结果点击后进入个股页面。
- 展示 StockContext。
- 展示行情、K 线、基本信息、行业、板块、是否在自选/持仓。
- 支持 AI 深研、追问、报告生成。

### 市场

市场页面承载大盘、板块和行业轮动：

- 大盘复盘。
- 板块热力图。
- 白酒、港股互联网、大型科技等板块。
- 板块关联标的。
- 事件时间线。

板块信息应放在市场页，而不是单独作为一级菜单。一级菜单保留给用户频繁操作对象。

### 盯盘

盯盘是独立的大功能模块：

- 盯盘开关。
- 规则列表。
- 事件队列。
- 通知策略。
- 噪声控制。
- 事件解释。
- 与个股、持仓、总览联动。

### 策略

策略页承载策略库和回测入口：

- 策略列表。
- 回测结果。
- 适用市场和标的。
- 风险提示。
- 与持仓和自选联动。

### 任务

任务页展示 agent 工作过程：

- 深研任务。
- 风控任务。
- 调仓规划任务。
- 失败重试。
- SSE 事件和步骤。

### 报告

报告页展示沉淀结果：

- 单股深研。
- 市场复盘。
- 风险报告。
- 调仓方案。
- 导出能力。

### 设置

设置页承载 Provider、Model、Skill、Tool、Profile 配置：

- Provider 配置。
- Model 路由。
- Skill 开关和权限。
- Tool 权限。
- 运行 profile。
- 审计和安全策略。

## AI Copilot 简化设计

AI Chat 需要轻量而明确：

- 顶部：当前上下文，例如页面、股票、权限、auto_trade 状态。
- 中部：对话消息和流式事件。
- 快捷问题：根据当前页面动态变化。
- 底部：输入框和发送按钮。

不需要在总览或顶部再放“AI 指挥台”。用户可以随时在右侧 Copilot 发起命令。

## Skill Trace 展示

右侧 Copilot 和任务页可以展示 `skill_trace`，但它的语义是“本次请求涉及哪些 Skills 和权限边界”，不是一个独立 Team Run。

展示规则：

- `skill_trace` 以事件或任务字段出现，展示 AI 研究员、AI 风控官、AI 调仓规划师、AI 报告员、AI 执行代理关闭占位。
- 每个 trace step 展示 `skill`、中文角色、状态、handoff、权限等级和说明。
- `execution-agent-disabled` 必须展示为 blocked/locked，并明确“真实交易保持关闭”。
- 调仓场景可以展示 `execution_guard.auto_trade=false`。
- 不在前端新增 Team Run 模式、Team Run 标签页或 `/api/team-runs` 调试入口。

推荐视觉位置：

- 右侧 Copilot：作为流式事件中的一类消息，跟随用户本次提问出现。
- 任务页：作为任务观察里的 Skill Trace 面板，和任务队列、SSE 事件并列展示。
- 设置页：继续展示 Skill 开关和锁定状态，不展示运行时 TeamRun 配置。

## 核心交互链路

```text
自选 -> 个股 -> 深研 -> 报告
持仓 -> 风险扫描 -> 调仓规划 -> 拟单草案
市场 -> 板块 -> 关联标的 -> 个股/自选
盯盘 -> 事件 -> 个股复核/持仓复核
Copilot -> IntentRouter -> SkillRegistry -> declarative skill_trace -> DeerFlow adapter -> 任务/报告/审计
```
