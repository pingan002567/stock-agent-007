# 菜单与信息架构

## 当前菜单顺序

左侧菜单最终排序：

```text
总览 / 自选 / 持仓 / 个股 / 市场 / 盯盘 / 策略 / 任务 / 报告
```

设置不作为普通菜单项，而是固定在左侧底部的齿轮图标。

## 排序理由

这套排序以用户资产工作流为主线：

1. 先看全局状态。
2. 再看自己关注的标的。
3. 再看真实持仓。
4. 再深入到个股。
5. 再用市场和板块解释变化。
6. 然后进入长期运行能力：盯盘、策略。
7. 最后查看过程和结果沉淀：任务、报告。

## 为什么“组合”改成“持仓”

“组合”偏投研或资产管理术语，可能包含模拟组合、策略组合、观察组合等含义。当前产品的核心是用户真实资产视角，因此“持仓”更直观：

- 用户能立刻理解。
- 与风险扫描、仓位规则、调仓规划直接关联。
- 更适合放在自选之后。

## 为什么新增“自选”

自选是用户还没持有但正在观察的对象池，也是 AI 主动服务的重要入口：

- 可以主动深研。
- 可以加入盯盘。
- 可以做同类比较。
- 可以转化为持仓决策。

自选和持仓共同构成用户的“关注范围”。

## 板块信息放在哪里

板块信息放在“市场”页，而不是新增一级菜单。

原因：

- 板块是市场解释语境，不是用户一级资产对象。
- 白酒、港股互联网、大型科技等板块应在市场页展示轮动、热度和关联标的。
- 个股页可以展示所属板块，但不负责承载全部板块信息。

## 个股搜索和展示放在哪里

个股搜索有两个层级：

- 顶部全局搜索：任何页面都可以搜代码、名称、拼音。
- 个股页：搜索结果点击后进入，展示完整 StockContext。

个股页承载：

- 行情和基本信息。
- K 线和历史走势。
- 所属行业和板块。
- 是否在自选。
- 是否在持仓。
- 是否开启盯盘。
- AI 深研、追问、报告。

## 菜单与页面联动

| 来源 | 目标 | 联动 |
| --- | --- | --- |
| 总览盯盘摘要 | 盯盘 | 查看全部事件和规则 |
| 总览持仓风险 | 持仓 | 进入风险扫描和调仓 |
| 自选股票 | 个股 | 打开 StockContext |
| 持仓股票 | 个股 | 打开 StockContext 并带仓位 |
| 市场板块 | 个股/自选 | 查看关联标的 |
| 盯盘事件 | 个股/持仓 | 做事件影响复核 |
| 策略结果 | 持仓/自选 | 生成候选调整 |
| Copilot 输出 | 任务/报告 | 沉淀过程和结论 |
